# Code Review

Use this skill when the task is to review a change, diff, pull request, or implementation.

Focus on concrete findings first:
- behavioral bugs
- safety or approval bypasses
- missing or weak tests
- compatibility risks

Report findings with file and line references when possible. Keep summaries secondary to the issues.
